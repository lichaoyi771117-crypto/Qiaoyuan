# 峤远修订规划 V1.0

> 日期：2026-07-11
> 状态：设计阶段，待用户批准后实施

---

## 一、背景与目标

峤远（F-Analyzer）当前为 MVP v0.3，已实现 Excel 报表解析 → 20+ 指标计算 → LLM 5 模块报告生成的基本流程。但存在以下短板：

1. **无主脑协议** — LLM system prompt 是通用财务分析师，缺乏 CFO 级专家框架
2. **API Key 是占位符** — 报告实际走 fallback 静态文本，LLM 分析跑不了
3. **上传体验粗糙** — 单文件上传，无 AI 分拣，多报表需手动合并
4. **报告太浅太短** — 摘要 200-300 字，诊断最多 5 条，无通俗化解释
5. **无客户需求校准** — 所有客户看同一份报告，无差异化
6. **无行业基准** — 参考值都是通用阈值，无行业对标
7. **成长能力全是 NaN** — 单期报表无法计算，但前端仍展示空值
8. **下载名不副实** — 标"PDF报告"实际下载 HTML
9. **首页无产品介绍** — 只有操作指引
10. **缺少优化改进建议** — 只诊断不开方，客户拿不到可落地的行动方案

**改造目标**：将峤远从"指标计算器"升级为"CFO 级 AI 财务诊断 + 优化顾问"。

## 二、峤远主脑协议 V1.0

### 2.1 设计来源

从楚曦玥 V9.3.2 提取分层架构框架 + Council 机制 + 咨询协议五步闭环，替换身份为 CFO 级财务专家，剥离双人格/写作插件/短视频插件等不相关模块，专业化为财务分析场景。

### 2.2 身份定义

**峤远 · 企业财务总监级 AI 专家**

- **四栈专家模型**：财务分析 + 税务筹划 + 融资顾问 + 法务合规
- **能力域**：报表解读、财务诊断、税务风险识别、融资能力评估、成本管控建议、内控管理、预算管理、报表优化
- **输出风格**：结论先行、数据支撑、通俗化解释（每个专业术语后跟一句大白话）

### 2.3 六条戒律

| 戒律 | 内容 | 来源映射 |
|------|------|---------|
| 1. 数据真实优先 | 所有分析必须基于上传报表的实际数字，不做推测，不编造数据 | 楚曦玥 CORE 3.3 冲突处理元规则 |
| 2. 风险穿透 | 不停留在表面指标，穿透到业务实质（如高负债率→是借款扩张还是经营亏损？） | 楚曦玥咨询协议 Step 4 IIEIA 框架 |
| 3. 行业基准锚定 | 指标评价必须对标行业基准，不做绝对化判断 | 楚曦玥 ARBITRATION 量化阈值 |
| 4. 趋势研判从严 | 单期数据不做趋势判断，明确标注局限性 | 楚曦玥 CORE 3.3 信息共享原则 |
| 5. 客户视角校准 | 分析维度必须匹配客户选择题校准的关注点 | 楚曦玥咨询协议 Step 3 客户视角校验 |
| 6. 可执行性统领 | 建议必须具体可操作，禁止"加强管理""提高效率"等空话 | 楚曦玥咨询协议 Step 5 方案输出 |

### 2.4 Council 机制（财务分析专用）

- **触发条件**：客户选择"融资能力评估"或 Altman Z-score < 1.81
- **六角色**：CFO 视角 / 税务师视角 / 融资顾问视角 / 内控审计视角 / 成本管控视角 / 反对者视角
- **规则**：每角色 ≤300 字，串行多角色模拟，必须暴露分歧

### 2.5 咨询协议五步闭环

| 步骤 | 名称 | 内容 |
|------|------|------|
| Step 1 | 需求结构化 | 选择题校准客户需求（行业/关注维度/报告用途） |
| Step 2 | 数据解析 | Excel 解析 + 指标计算 + 质量校验 |
| Step 3 | 客户视角校验 | 代入客户角色审计分析方案 |
| Step 4 | 逻辑推演 | IIEIA 框架（Issue→Insight→Evidence→Implication→Action） |
| Step 5 | 方案输出 | 分析报告 + 优化建议 + 风险提示 |

### 2.6 输出文件

`docs/峤远主脑协议_V1.0.md`（实施阶段新建）

## 三、智能上传 + AI 分拣

### 3.1 复用程晓融机制，适配 Streamlit

| 对比项 | 程晓融（FastAPI） | 峤远改造后（Streamlit） |
|--------|------------------|----------------------|
| 上传方式 | 拖拽框 + 手动上传可折叠 | st.file_uploader(accept_multiple_files=True) |
| 分拣方式 | /smart-classify API 端点 | 内联 LLM 调用，对每个文件前 20 行预览分类 |
| 分类标签 | 个人/企业各类证件 | 资产负债表/利润表/现金流量表/附注/其他 |
| 展示方式 | 文件列表 + 类型标签 | 文件列表 + 类型标签 + 识别置信度 |

### 3.2 流程

1. 用户拖拽多个文件到上传区
2. 系统读取每个文件前 20 行，发送给 LLM 分类
3. 显示分类结果（如：`📊 资产负债表.xlsx → 资产负债表 ✓`）
4. 合并所有财务报表数据进入分析管线
5. 原"分析目标"多选框替换为下方的选择题校准

## 四、单轮选择题校准

### 4.1 设计原则

参考乔曦双轮机制，简化为一轮（3 题）。客户可选择跳过，输出标准版报告。

### 4.2 三道选择题

| 题号 | 问题 | 类型 | 选项 | 作用 |
|------|------|------|------|------|
| Q1 | 您的企业所属行业？ | 单选 | 制造业/建筑业/批发零售/餐饮服务/科技信息/农林牧渔/交通运输/其他 | 决定行业基准对标数据 |
| Q2 | 您最关注的财务维度？ | 多选 ≤3 | 偿债能力/盈利能力/营运效率/现金流健康/税务合规/融资能力/成本管控 | 决定报告侧重点、指标展示优先级、Council 是否触发 |
| Q3 | 报告主要用途？ | 单选 | 内部管理/银行贷款/投资融资/税务规划/总体体检(默认) | 决定报告语气和深度 |

### 4.3 跳过机制

"跳过选择题，直接分析"按钮 → 输出标准版报告（所有维度均衡展示，通用行业基准）

### 4.4 校准对报告的影响

- **Q1（行业）** → 选择对应行业的基准值（制造业/建筑业/批发零售等各有不同基准）
- **Q2（关注维度）** → 调整指标卡片展示顺序、报告各模块篇幅权重、**抽取针对性增强指标（见第五章）**、触发 Council
- **Q3（报告用途）** → 银行贷款侧重偿债和稳定性；投资融资侧重成长性和盈利能力；税务规划侧重税务风险

## 五、针对性指标抽取机制（核心差异化引擎）

### 5.1 设计思路

> 默认指标是"标准体检套餐"，针对性指标是"专项深度检查"。

峤远当前能计算 7 大类 20 个指标，但所有客户看到的都是同样的一屏数字。科目映射表里有 47 个数据字段，其中大量字段（如销售费用、短期借款、应付账款等）只被用来做中间计算，没有单独输出为指标。

**改造逻辑**：
1. 将指标分为**基础指标**（所有报告都展示）和**增强指标**（根据 Q2 选择抽取）
2. 增强指标由现有数据字段派生，不需要额外的数据输入
3. 客户选了哪个关注维度，就额外计算并展示该维度的增强指标
4. LLM 在报告中对增强指标做深度解读，而非一笔带过

### 5.2 指标分类总览

#### A. 基础指标（默认展示，共 17 个）

| 大类 | 指标 | 当前状态 |
|------|------|---------|
| 偿债能力 | 流动比率、速动比率、资产负债率、利息保障倍数 | ✅ 已有 |
| 盈利能力 | 毛利率、净利率、ROE、ROA、EBITDA率 | ✅ 已有 |
| 营运效率 | 存货周转天数、应收账款周转天数、总资产周转率 | ✅ 已有 |
| 现金流 | 经营现金流/净利润、自由现金流 | ✅ 已有 |
| 税务指标 | 实际税率、综合税负率 | ✅ 已有 |
| 财务预警 | Altman Z-score | ✅ 已有 |

> 注：成长能力 3 个指标因单期报表无法计算，从基础展示中移除，改为"需多期数据"提示。

#### B. 增强指标库（按 Q2 关注维度分类，共 25 个新增）

##### B1. 偿债能力 — 增强指标（4 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 现金比率 | 货币资金 / 流动负债 | cash_and_equivalents, total_current_liabilities | 最严格的偿债测试，排除应收和存货变现风险 |
| 产权比率 | 总负债 / 所有者权益 | total_liabilities, total_equity | 资本结构是否稳健，银行关注的核心指标 |
| 营运资金 | 流动资产 - 流动负债 | total_current_assets, total_current_liabilities | 绝对值口径的短期偿债缓冲垫 |
| 有息负债率 | (短期借款+长期借款) / 总资产 | short_term_borrowings, long_term_borrowings, total_assets | 区分"有息负债"和"经营性负债"，真实杠杆水平 |

##### B2. 盈利能力 — 增强指标（3 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 营业利润率 | 营业利润 / 营业收入 | operating_income, revenue | 主业赚钱能力，剔除营业外收支干扰 |
| 成本费用利润率 | 利润总额 / (营业成本+税金及附加+三费+研发) | total_profit, cost_of_goods_sold, tax_and_surcharge, selling_expenses, administrative_expenses, rd_expenses, financial_expenses | 每花1块钱成本费用能赚多少利润，综合效率指标 |
| 非经常性损益占比 | (营业外收入-营业外支出) / 利润总额 | non_operating_income, non_operating_expense, total_profit | 利润含金量，占比高说明靠非经常性收入撑利润 |

##### B3. 营运效率 — 增强指标（3 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 应付账款周转天数 | 365 / (营业成本 / 应付账款) | cost_of_goods_sold, accounts_payable | 对供应商的议价能力，占用上游资金的能力 |
| 现金周转周期 | 存货周转天数 + 应收账款周转天数 - 应付账款周转天数 | 已有指标组合计算 | 资金从投入到回收的全周期，越短越好 |
| 固定资产周转率 | 营业收入 / 固定资产 | revenue, fixed_assets | 固定资产产出效率，重资产行业关键指标 |

##### B4. 现金流健康 — 增强指标（5 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 现金流量比率 | 经营活动现金流净额 / 流动负债 | operating_cash_flow, total_current_liabilities | 用现金流而非资产来衡量短期偿债能力 |
| 经营现金流/营业收入 | 经营活动现金流净额 / 营业收入 | operating_cash_flow, revenue | 每赚1块钱收入有多少变成真金白银 |
| 资本支出/经营现金流 | 资本支出 / 经营活动现金流净额 | capital_expenditure, operating_cash_flow | 扩张是否过度，自由现金流是否被投资吃掉 |
| 投资活动现金流 | 直接取值 | investing_cash_flow | 是在扩张（负值）还是在收缩/变卖（正值） |
| 筹资活动现金流 | 直接取值 | financing_cash_flow | 是在融资（正值）还是在还债/分红（负值） |

##### B5. 税务合规 — 增强指标（3 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 所得税费用率 | 所得税费用 / 营业收入 | income_tax_expense, revenue | 所得税负担是否合理 |
| 间接税负担率 | 税金及附加 / 营业收入 | tax_and_surcharge, revenue | 增值税附加+消费税等间接税负担 |
| 应交税费占比 | 应交税费 / 营业收入 | tax_payable, revenue | 欠税规模相对收入的比重 |

##### B6. 融资能力 — 增强指标（4 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 产权比率 | 总负债 / 所有者权益 | total_liabilities, total_equity | 杠杆水平，银行判断放贷空间的核心 |
| 有息负债率 | (短期借款+长期借款) / 总资产 | short_term_borrowings, long_term_borrowings, total_assets | 真实付息杠杆，区分经营性负债 |
| 银行融资依赖度 | (短期借款+长期借款) / 总负债 | short_term_borrowings, long_term_borrowings, total_liabilities | 对银行借款的依赖程度，越高再融资越难 |
| 净资产融资支撑度 | 所有者权益 / (短期借款+长期借款) | total_equity, short_term_borrowings, long_term_borrowings | 每元借款有多少净资产做后盾 |

> Altman Z-score（已有）在融资能力维度中作为综合预警指标重点解读。

##### B7. 成本管控 — 增强指标（6 个）

| 指标 | 公式 | 数据来源字段 | 业务意义 |
|------|------|-------------|---------|
| 销售费用率 | 销售费用 / 营业收入 | selling_expenses, revenue | 销售投入效率，是否花钱买收入 |
| 管理费用率 | 管理费用 / 营业收入 | administrative_expenses, revenue | 管理效率，机构是否臃肿 |
| 研发费用率 | 研发费用 / 营业收入 | rd_expenses, revenue | 创新投入力度，科技企业关注 |
| 财务费用率 | 财务费用 / 营业收入 | financial_expenses, revenue | 融资成本负担 |
| 期间费用率 | (销售+管理+研发+财务费用) / 营业收入 | 四费合计, revenue | 综合费用管控水平 |
| 营业成本率 | 营业成本 / 营业收入 | cost_of_goods_sold, revenue | 成本控制能力，与毛利率互补 |

### 5.3 Q2 选择 → 增强指标抽取映射

| Q2 选择的关注维度 | 抽取的增强指标 | 新增个数 |
|-----------------|--------------|---------|
| 偿债能力 | 现金比率、产权比率、营运资金、有息负债率 | +4 |
| 盈利能力 | 营业利润率、成本费用利润率、非经常性损益占比 | +3 |
| 营运效率 | 应付账款周转天数、现金周转周期、固定资产周转率 | +3 |
| 现金流健康 | 现金流量比率、经营现金流/营业收入、资本支出/经营现金流、投资活动现金流、筹资活动现金流 | +5 |
| 税务合规 | 所得税费用率、间接税负担率、应交税费占比 | +3 |
| 融资能力 | 产权比率、有息负债率、银行融资依赖度、净资产融资支撑度 | +4 |
| 成本管控 | 销售费用率、管理费用率、研发费用率、财务费用率、期间费用率、营业成本率 | +6 |

> 客户最多选 3 个维度，因此最多新增 15 个增强指标（如选现金流+成本管控+营运效率 = +14 个），加上 17 个基础指标，报告最多展示 32 个指标。
>
> 部分指标跨维度复用（如产权比率同时在偿债和融资中出现），实际不重复计算，只在不同维度报告中从不同角度解读。

## 六、经典财务分析模型固化（3模型 · 默认报告标配）

### 6.1 设计思路

> 指标是"零件"，模型是"装配线"。三个经典模型分别回答企业经营的核心三问：

| 模型 | 核心问题 | 通俗版 | 状态 |
|------|---------|--------|------|
| 杜邦分析 | ROE 为什么是这个数？ | "你的赚钱效率拆开来看，哪块强哪块弱？" | **新增** |
| Altman Z-Score | 企业会不会倒？ | "你的企业离破产有多远？" | ✅ 已有 |
| Piotroski F-Score | 基本面在变好还是变差？ | "你的企业是在走上坡路还是下坡路？" | **新增** |

三个模型互补不重叠：杜邦看结构（横截面），Z-score 看风险（底线），F-Score 看趋势（方向感）。

### 6.2 模型一：杜邦分析（DuPont Analysis）

#### 公式

**经典三因素分解**：
```
ROE = 净利率 × 总资产周转率 × 权益乘数
    = (净利润/营业收入) × (营业收入/总资产) × (总资产/所有者权益)
```

#### 分解树图示

```
                    ┌─────────┐
                    │  ROE    │  12.5%
                    └────┬────┘
              ┌──────────┼──────────┐
              │          │          │
        ┌─────┴─────┐ ┌──┴───┐ ┌───┴────┐
        │  净利率   │ │ 资产  │ │ 权益   │
        │          │ │ 周转率│ │ 乘数   │
        │  8.3%    │ │ 1.5次 │ │ 1.0倍  │
        └─────┬─────┘ └──┬───┘ └───┬────┘
              │          │          │
         ┌────┴────┐ ┌───┴────┐ ┌───┴────┐
         │净利润   │ │营业收入│ │总资产  │
         │100万    │ │1200万  │ │1200万  │
         ├─────────┤ ├────────┤ ├────────┤
         │营业收入 │ │总资产  │ │权益    │
         │1200万   │ │1200万  │ │1200万  │
         └─────────┘ └────────┘ └────────┘
```

#### 单期 vs 双期模式

| 模式 | 数据要求 | 输出内容 |
|------|---------|---------|
| **静态分解**（1年期） | 当期报表 | 分解树图 + 各因子解读 + "你的ROE主要靠哪一块驱动" |
| **动态对比**（2年期） | 当期 + 上年同期 | 并排分解树 + 各因子变化值 + "今年ROE变动主要由XX因子驱动" |

> **对用户意见的评估**：用户认为杜邦分析"需要至少2年财报才能开展"。实际上，三因素分解本身只需1年数据即可计算和画图。但2年数据能做趋势对比（哪个因子变好了/变差了），分析价值显著提升。因此采用双模式设计：1年也能做，2年更深入。不强制要求2年。

#### 画图方案

Streamlit 原生支持 `st.graphviz_chart()`，用 Graphviz 画分层树状图：
- 每个节点显示指标名称 + 数值
- 节点颜色：绿色（优于行业基准）、黄色（接近基准）、红色（偏离基准）
- 双期模式下，并排两棵树，中间用箭头标注变化方向

### 6.4 模型三：Piotroski F-Score 财务质量评分（新增）

#### 公式

9 项二元信号（满足=1分，不满足=0分），总分 0-9 分：

| 维度 | 编号 | 信号 | 判断条件 | 数据需求 |
|------|------|------|---------|---------|
| **盈利能力** | F1 | ROA > 0 | 净利润/总资产 > 0 | 单期可算 |
| | F2 | 经营现金流为正 | 经营活动现金流净额 > 0 | 单期可算 |
| | F3 | ROA 同比提升 | 当期ROA > 上期ROA | **需双期** |
| | F4 | 现金流 > 净利润 | 经营现金流 > 净利润（盈余质量） | 单期可算 |
| **杠杆与流动性** | F5 | 长期负债率下降 | 当期 < 上期 | **需双期** |
| | F6 | 流动比率提升 | 当期 > 上期 | **需双期** |
| | F7 | 未增发新股 | 当期实收资本 ≤ 上期 | **需双期** |
| **营运效率** | F8 | 毛利率提升 | 当期 > 上期 | **需双期** |
| | F9 | 资产周转率提升 | 当期 > 上期 | **需双期** |

#### 单期 vs 双期模式

| 模式 | 可算项 | 评分范围 | 说明 |
|------|--------|---------|------|
| **单期模式** | F1, F2, F4 | 0-3 分 | 仅盈利能力维度，标注"完整评分需上传上年报表" |
| **完整模式** | F1-F9 | 0-9 分 | 全部9项，标准判据 |

判据（完整模式）：
- 7-9 分：🟢 财务状况持续改善
- 4-6 分：🟡 财务状况平稳
- 0-3 分：🔴 财务状况恶化

#### 非上市企业适配

- F7"未增发新股" → 改为"实收资本未增加"（检查 paid_in_capital 是否变化）
- 其余8项均基于会计数据，无需市值，天然适用非上市企业

#### 代码实现

新建 `src/piotroski_scorer.py`：
```python

    """Piotroski F-Score 财务质量评分"""

    def score(self, current_df, prior_df=None) -> dict:
        """
        单期: 返回 {f1,f2,f4, partial_score: 0-3, note: "完整评分需上年数据"}
        双期: 返回 {f1-f9, total_score: 0-9, dimension_scores: {盈利,杠杆,效率}}
        """
```

### 6.5 模型解释固化语言（报告标配 · 客户教育）

#### 固化文本一：杜邦分析（DuPont Analysis）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 模型一：杜邦分析 — 你的赚钱效率拆开来看
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【这是什么？】
杜邦分析是杜邦公司发明的一种财务分解方法，把"净资产收益率（ROE）"——
也就是你每投入1块钱本金能赚多少——拆成三个零件，让你看清赚钱的来源。

ROE = 净利率 × 资产周转率 × 权益乘数

把这三个零件翻译成大白话：
  • 净利率 = 每卖100块钱东西，到手能赚多少利润 → 衡量"卖得贵不贵"
  • 资产周转率 = 你的资产一年能转几圈 → 衡量"卖得快不快"
  • 权益乘数 = 你借了多少钱来做生意 → 衡量"杠杆大不大"

【怎么读？】
看分解树图，三个零件哪个高哪个低，就知道你的ROE主要靠什么驱动：
  • 如果靠净利率高 → 你走的是"高利润率路线"（产品有定价权）
  • 如果靠周转率高 → 你走的是"薄利多销路线"（靠规模和效率）
  • 如果靠权益乘数高 → 你走的是"杠杆路线"（靠借钱放大收益）
    ⚠️ 注意：杠杆是双刃剑，赚钱时放大收益，亏钱时也放大亏损

【你应该关注什么？】
1. 三个零件哪个最强、哪个最弱 → 知道该补哪块
2. 最弱的那个就是你的"提升杠杆点" → 优化它对ROE的提升效果最大
3. 如果权益乘数过高（比如>3）→ 赚钱靠借钱，风险在积聚
4. 如果有上年数据 → 看哪个零件变了，是变好的还是变坏的
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【这是什么？】
Z-Score是纽约大学Edward Altman教授发明的破产预测模型，用5个财务指标
算出一个综合分数，被全球银行和投资机构广泛使用。它的准确率在预测

【怎么读？】

  🟢 Z > 2.99  → 安全区：企业财务健康，短期无破产风险

【你应该关注什么？】
1. Z值落在哪个区域 → 整体风险定位
2. 5个分变量哪个拖了后腿 → 那个就是最需要改善的

   X2 留存收益/总资产 → 历史积累能力
   X3 EBIT/总资产 → 经营盈利能力
   X4 权益/总负债 → 资本结构稳健性
   X5 营业收入/总资产 → 资产产出效率
3. 如果在灰色区或危险区 → 必须优先改善最弱的1-2个分变量
4. Z值不是越高越好 → 过高的Z值可能意味着资产闲置、未充分利用杠杆
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

#### 固化文本三：Piotroski F-Score（财务质量评分）

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📌 模型三：Piotroski F-Score — 你的企业在走上坡路还是下坡路？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

【这是什么？】
F-Score是芝加哥大学Joseph Piotroski教授发明的财务质量评分模型，
用9个"是/否"信号来衡量企业财务状况是在变好还是变差。每满足一项得
1分，满分9分。它不看你"现在好不好"，而是看你"方向对不对"。

【怎么读？】
看你的总分落在哪个区间：
  🟢 7-9分 → 财务状况持续改善，走上坡路
  🟡 4-6分 → 财务状况基本平稳，没有明显改善或恶化
  🔴 0-3分 → 财务状况在恶化，走下坡路，需要警惕

9项信号分三大块：
  【盈利能力】F1 有没有赚钱（ROA>0）
              F2 有没有真金白银（经营现金流>0）
              F3 比去年赚得多吗（ROA同比提升）
              F4 赚的钱是不是真钱（现金流>净利润）
  【杠杆与流动性】F5 负债有没有减少
                  F6 短期偿债能力有没有变强
                  F7 有没有稀释股东权益（增发新股）
  【营运效率】F8 毛利率有没有提升
              F9 资产周转有没有变快

【你应该关注什么？】
1. 总分落在哪个区间 → 你的企业"方向感"如何
2. 失分的项在哪 → 那些就是正在变差的方面
3. F3/F5/F6/F7/F8/F9需要上年数据才能算 → 单期只能评0-3分
   ⚠️ 如果只上传了1期报表，F-Score只有部分评分（满分3分），
      建议上传上年报表获取完整9分评分
4. F4（现金流>净利润）失分要特别警惕 → 账上有利润但收不回钱，
   是中小企业最常见的"纸面富贵"陷阱
5. F-Score和Z-Score要一起看：
   Z-Score看"你现在安不安全"，F-Score看"你在往哪走"
   Z值高+F-Score高 = 最理想：安全且在变好
   Z值高+F-Score低 = 需警惕：现在还行但在走下坡路
   Z值低+F-Score高 = 有希望：虽然不安全但在改善
   Z值低+F-Score低 = 最危险：不安全且还在恶化
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 固化文本四：三模型联立解读（模型解释之后的数据解读）

模型解释文本输出后，紧接着是客户自己的实际数据和分析结论。这部分由 LLM 生成，但 prompt 中固化以下结构要求：

```
请基于客户实际数据，对三个模型进行联立解读，必须包含：

1. 【杜邦分析】客户ROE数值 → 三因素分解 → 主要驱动因子 → 改进建议
   格式：你的ROE是X%，拆开来看：净利率X%（高/中/低）、资产周转率X次
   （快/中/慢）、权益乘数X倍（高/中/低）。你的ROE主要靠XX驱动，
   XX是短板，建议从XX入手改进。

2. 【Z-Score】客户Z值 → 所处区域 → 最弱分变量 → 改善方向
   格式：你的Z值是X，处于XX区域。5个分变量中XX得分最低，意味着…
   建议优先改善XX。

3. 【F-Score】客户F值 → 所处区间 → 失分项 → 警示
   格式：你的F-Score是X分，财务状况在XX。失分的项是XX，说明…
   特别注意F4（盈余质量）的得分情况。

4. 【三模型联立】综合判断
   格式：综合三个模型来看，你的企业目前XX，正在XX，建议XX。
```

## 七、优化改进建议模块（新增）

### 7.1 设计思路

> 光诊断不开方，等于体检完不给处方。峤远作为 CFO 级 AI 专家，输出的不仅仅是"指标好不好"，更要给出"怎么变好"。

在报告的诊断模块之后，新增**优化改进建议模块**，根据选择题校准的客户需求 + 指标计算结果（含增强指标），输出针对性的行动方案。

### 7.2 四大建议维度

| 维度 | 触发条件 | 建议内容示例 |
|------|---------|-------------|
| **报表优化** | 质量校验发现异常（负值/极值/逻辑不平衡） | "应收账款占总资产 45%，建议按账龄计提坏账准备，并在附注中披露前五大客户集中度" |
| **财务管理** | 偿债/盈利/营运指标偏离基准 | "流动比率 0.8 低于行业基准 1.5，建议：①缩短应收账款账期至 30 天；②与供应商协商延长应付账款至 60 天；③建立 3 个月运营资金储备" |
| **成本管控** | 毛利率/期间费用率异常 | "销售费用率 18% 高于行业 8%，建议：①拆分费用明细核查广告投放 ROI；②引入费用预算月度审批机制；③对差旅费实行人均定额管控" |
| **税收筹划** | 税负率偏高/偏低、税务风险指标异常 | "增值税税负率 1.2% 低于行业 3.5%，存在被稽查风险。建议：①核查进项税抵扣是否合规；②评估简易计税适用性；③留存供应链发票完整性证据链" |

### 7.3 建议生成逻辑

```
指标计算结果（基础17 + 增强N）
  ├── 指标偏离基准 → 触发"财务管理"建议
  ├── 质量校验异常 → 触发"报表优化"建议
  ├── 成本费用率异常 → 触发"成本管控"建议
  └── 税负率异常 → 触发"税收筹划"建议
      ↓
  叠加客户选择题校准的 Q2 关注维度 + 增强指标深度数据
      ↓
  LLM 生成针对性建议（每条 ≤200 字，必须含：问题→原因→措施→预期效果）
```

### 7.4 建议输出格式

每条建议遵循 **问题→原因→措施→预期效果** 四段式：

> **【成本管控】销售费用率偏高**
> - **问题**：销售费用率 18%，高于行业基准 8.5%
> - **原因**：可能存在广告投放效率低、销售人员人均产出不足、差旅费管控缺失
> - **措施**：①建立广告投放 ROI 月度评估机制，砍掉 ROI < 1 的渠道；②将销售费用预算拆分至产品线，按产出比分配；③差旅费实行人均定额制，超额需总监审批
> - **预期效果**：预计 6 个月内销售费用率可降至 12-14%，年节约费用约 XX 万元

## 八、双版本报告机制（专业版 + 大白话版）

### 8.1 设计思路

> 同一份体检数据，老板要的是"有没有问题、怎么办"，财务总监要的是"指标细节、验证逻辑"。两个角色的信息需求完全不同，一份报告不可能同时满足。

峤远同时输出两个版本的报告，基于同一套指标计算结果和模型分析数据，但采用不同的语言体系、深度和结构。客户可以自由选择在线查看哪个版本、下载哪个版本，也可以两个都要。

### 8.2 两个版本对比

| 维度 | 版本1：专业版 | 版本2：大白话版 |
|------|-------------|---------------|
| **受众** | 财务总监 / 财务人员 / 银行信贷员 | 老板 / 个体户 / 非财务背景决策者 |
| **语言** | 标准财务术语，保留专业表达 | 每个术语后跟一句大白话翻译，用类比和比喻 |
| **指标展示** | 完整指标值 + 公式 + 行业基准对标值 + 偏差幅度 | 只展示关键数字 + 红/黄/绿灯 + 一句话结论 |
| **模型分析** | 完整分解树/仪表盘/条形图 + 5个Z分变量 + 9项F信号明细 | 只展示模型结论 + 红/黄/绿灯 + "你的企业在…"一句话判断 |
| **优化建议** | 四段式（问题→原因→措施→预期效果），措施具体到操作步骤 | 三段式（做什么→为什么→怎么做），口语化，突出行动项 |
| **报告结构** | 7模块完整展开（摘要→诊断→模型→建议→税务→融资→免责） | 3段式精简（体检结论→重点关注→行动清单） |
| **篇幅** | 5000-8000字 | 2000-3500字 |
| **语气** | 客观、专业、数据驱动 | 直接、通俗、结论先行 |
| **数据表** | 完整数据表格（指标/值/基准/偏差/评级） | 无表格或极简表格（只有红黄绿灯） |

### 8.3 版本1：专业版财务分析报告

#### 报告结构（7模块）

| 模块 | 内容 | 篇幅要求 |
|------|------|---------|
| 1. 执行摘要 | 企业财务状况概览 + 关键指标一览表 + 总体评级 | ≥400字 |
| 2. 财务诊断 | 基础17指标 + 增强指标逐项深度解读（含公式、行业基准、偏差分析） | ≥1500字 |
| 3. 经典模型分析 | 模型解释固化文本 + 杜邦分解树 + Z-Score 5分变量 + F-Score 9项信号 + 联立解读 | ≥800字 |
| 4. 优化改进建议 | 4维度建议（报表优化/财务管理/成本管控/税收筹划），四段式输出 | ≥1000字 |
| 5. 税务分析 | 税负率分析 + 税务风险识别 + 合规建议 | ≥400字 |
| 6. 融资能力评估 | Council多角色分析（触发条件：Q2选"融资能力"或Z<1.81） | ≥500字（触发时） |
| 7. 免责声明 | 数据来源说明 + 分析局限性 + 使用边界 | 固定文本 |

#### 语言风格示例

```

流动比率 0.85，低于行业基准 1.5（制造业），偏差 -43.3%，评级：⚠️ 偏低。
速动比率 0.52，低于行业基准 1.0，偏差 -48.0%，评级：⚠️ 偏低。
上述两项指标表明企业短期偿债能力显著不足，流动资产对流动负债的覆盖
能力较弱。结合资产负债率 68.5%（高于行业 55%），综合判断企业存在
短期流动性风险，建议通过加速应收账款回笼、优化存货管理、适度延长
应付账款账期等方式改善营运资金结构。

Z-Score = 1.72，处于危险区域（<1.81），5个分变量中：
  X1（营运资金/总资产）= -0.03，拖累明显
  X2（留存收益/总资产）= 0.08，积累不足
  X3（EBIT/总资产）= 0.12，经营盈利尚可
  X4（权益/总负债）= 0.46，资本结构偏弱
  X5（营业收入/总资产）= 0.85，资产产出效率中等
主要风险点在X1（营运资金为负）和X4（负债率偏高），建议优先改善
营运资金管理。
```

### 8.4 版本2：大白话版报告

#### 报告结构（3段式）

| 段落 | 内容 | 篇幅要求 |
|------|------|---------|
| **一、体检结论** | 一段话总评（你的企业整体财务健不健康）+ 红/黄/绿灯总览表 | ≥300字 |
| **二、重点关注** | 最多5个问题点，每个用"有问题吗→严不严重→什么原因→不管会怎样"四问展开 | ≥1000字 |

#### 语言风格示例

```

总体来看，你的企业目前属于"亚健康"状态——还没到大问题，但有几个
信号灯在闪黄光，不管的话可能变成红灯。

  偿债能力：🟡 黄灯（短期还钱有点吃力）
  盈利能力：🟢 绿灯（还在赚钱，但赚得不多）
  营运效率：🔴 红灯（钱转得太慢，压在库存和应收里）
  现金流：🟡 黄灯（有利润但收回来的钱不够多）
  税务合规：🟢 绿灯（税负正常）

【重点关注 · 第1个问题】
  ❓ 有没有问题？——有。你的流动比率只有0.85，意思是手里能很快
     变现的钱只够还85%的短期欠款，剩下的15%还不上。
  ⚠️ 严不严重？——黄灯偏红。行业平均是1.5（手里有1块5能还1块钱
     的债），你只有8毛5。
  🔍 什么原因？——主要是两个：一是应收账款收得慢（客户欠你钱不还），
     二是存货压得太多（货卖不出去压在仓库里）。
  📉 不管会怎样？——短期欠款到期还不上，轻则影响企业信用，重则
     被供应商断货、被银行抽贷。

【行动清单 · 第1项】
  ✅ 做什么：把应收账款回收天数从现在的90天压到60天以内
  💡 为什么：客户欠你的钱每多压一天，你的资金成本就多一天
  🛠️ 怎么做：①给30天内付款的客户2%折扣；②超60天的客户停止供货；
     ③派专人每周跟进一次催收
  ⏰ 多久见效：3个月内流动比率可从0.85提升到1.1左右

### 8.5 技术实现方案

#### LLM 调用策略

两个版本共享同一套指标数据和模型分析结果，但使用**不同的 system prompt 和 user prompt** 分别调用 LLM 生成：

```
指标计算 + 模型分析（共享数据）
        │
    ┌───┴───┐
    │       │
  专业版    大白话版
  prompt    prompt
    │       │
  LLM调用   LLM调用
    │       │
  专业版    大白话版
  报告      报告

- 专业版：`_PROFESSIONAL_SYSTEM_PROMPT`（主脑协议驱动，专业术语，7模块）

两个 prompt 都注入相同的指标数据、模型数据、客户校准信息，但对输出格式和语言的要求完全不同。

#### report_generator.py 改造

```python
class ReportGenerator:

    def generate(self, metrics, model_results, calibration, prior_metrics=None):
        """生成双版本报告"""
        shared_context = self._build_shared_context(metrics, model_results, calibration, prior_metrics)

        professional = self._generate_professional(shared_context)
        plain = self._generate_plain(shared_context)

        return {
            "professional": professional,  # 专业版完整markdown
            "plain": plain,                # 大白话版完整markdown
        }

    def _generate_professional(self, context):
        """专业版：7模块，专业术语，深度分析"""
        prompt = self._build_professional_prompt(context)
        return self._call_llm(self._PROFESSIONAL_SYSTEM_PROMPT, prompt)

    def _generate_plain(self, context):
        """大白话版：3段式，通俗化，红黄绿灯"""
        prompt = self._build_plain_prompt(context)
        return self._call_llm(self._PLAIN_SYSTEM_PROMPT, prompt)
```

#### 大白话版 system prompt 核心铁律（_PLAIN_SYSTEM_PROMPT）

```
你是峤远，一个专门给老板讲财务的AI助手。你的读者是没学过财务的
小老板和个体户。

铁律：
1. 每个专业术语后面必须跟一句大白话翻译
   ✅ "流动比率0.85（手里能很快变现的钱只够还85%的短期欠款）"
   ❌ "流动比率0.85，低于行业基准"
2. 不用公式，只说结论和含义
   ✅ "你的赚钱效率12.5%，拆开看：卖得贵不贵(偏低)、卖得快不快(不错)、借了多少钱(很少)"
   ❌ "ROE = 净利率 × 资产周转率 × 权益乘数 = 8.3% × 1.5 × 1.0"
3. 每个问题用"四问"展开：有没有问题？严不严重？什么原因？不管会怎样？
4. 每个建议用"四答"展开：做什么？为什么？怎么做？多久见效？
5. 用红/黄/绿灯代替数字评级
6. 禁止出现：同比、环比、权益乘数、EBITDA、边际贡献等术语
   （如必须使用，立即在括号内翻译）
7. 结论先行：每段第一句话就是结论，后面才是解释
8. 语气像一个懂行的老朋友在聊天，不像在念报告
```

#### 专业版 system prompt 核心要求（_PROFESSIONAL_SYSTEM_PROMPT）

```
你是峤远，企业财务总监级AI专家。你的读者是财务总监、财务经理或
银行信贷员，他们有专业基础，需要通过报告验证自己的判断。

要求：
1. 使用标准财务术语，保留专业表达
2. 每个指标必须展示：数值 + 计算公式 + 行业基准 + 偏差幅度 + 评级
3. 模型分析必须展示完整分解树/分变量/信号明细
4. 优化建议按"问题→原因→措施→预期效果"四段式，措施具体到操作步骤
5. 引用主脑协议六条戒律作为分析逻辑依据
6. 数据来源和计算过程可追溯
7. 语气客观、专业、数据驱动
```

### 8.6 两个版本的数据一致性保证

两个版本基于同一套计算结果，数据一致性由以下机制保证：

1. **共享数据层**：指标计算、模型分析只算一次，两个版本共用
2. **固化文本层**：模型解释固化文本在专业版中完整展示，在大白话版中替换为简化版
3. **LLM约束**：两个 prompt 都注入相同的数据JSON，禁止 LLM 自行编造或修改数据
4. **数据校验**：生成后自动比对两个版本中引用的关键数字是否一致

### 8.7 对报告模块结构的影响

| 模块 | 专业版 | 大白话版 |
|------|--------|---------|
| 执行摘要 | 模块1（≥400字，含指标一览表） | 段落一·体检结论（≥300字，含红黄绿灯总览） |
| 财务诊断 | 模块2（≥1500字，逐项深度解读） | 段落二·重点关注（≥1000字，最多5个问题四问展开） |
| 经典模型分析 | 模块3（≥800字，完整分解+分变量+信号） | 段落二内嵌（简化结论+红黄绿灯，无分解树） |
| 优化改进建议 | 模块4（≥1000字，四段式） | 段落三·行动清单（≥700字，四答式） |
| 税务分析 | 模块5（≥400字） | 段落二内嵌（如有税务问题则展开） |
| 融资能力评估 | 模块6（≥500字，Council） | 段落二内嵌（简化结论） |
| 免责声明 | 模块7（固定文本） | 段末（简化版固定文本） |
