## 一、用户画像：我们为谁而做？

### 1.1 市场的断层

中国有 **5,200 万家中小微企业**，其中绝大多数存在一个共同的困境：

- 📌 **报表有人做，没人看**——代账会计每月按时出报表，但老板拿到手只有几张 Excel，不知道那些数字代表什么、意味着什么
- 📌 **财务指标看不懂**——流动比率、速动比率、资产负债率、EBITDA……大多数中小企业主并非财务出身，看不懂这些专业指标
- 📌 **融资卡在报表上**——去银行申请贷款，银行经理问"你 ROE 多少？利息保障倍数怎么样？"，老板答不上来
- 📌 **请不起专职财务总监**——一个好 CFO 年薪 30-50 万，中小企业养不起，但财务分析又是刚需
- 📌 **代账公司只管"记"不管"析"**——全国几十万家代账公司主要做账+报税，财务分析不在服务范围内

## 三、技术实现：怎么做到的？

### 3.1 技术架构概要

峤远采用**"全本地计算 + AI 云端解读"**的混合架构：

```
用户上传财务报表 Excel（.xlsx / .xls）
   → 自研 Excel 解析器（自动识别 6 种真实企业报表格式）
   → 自研 AccountingMapper（80+ 中国会计准则科目 → 标准字段映射）
   → 自研 QualityChecker（4 项数据质量校验：平衡/负值/逻辑/极值）
   → 自研 MetricCalculator（7 大类 20+ 财务指标，100% 自研公式）
   → DeepSeek API 生成专业解读报告（仅发送结构化指标摘要）
   → Streamlit 前端展示（14 个指标卡片 + 5 模块报告 + HTML/CSV 下载）
```

### 3.2 核心技术难点

**难点一：中小企业非标报表的自适应解析**

中国中小企业的 Excel 报表格式千奇百怪——有的用 2 列表格（科目名称 + 期末余额），有的用 6 列左右对照（含期初/期末），有的用 8 列带行次编号。科目名称也不统一——同一项"应收账款"，有的写"应收"，有的写"应收账款净额"，有的带括号备注。

峤远的解析器自动检测 6 种真实格式变体：行次编号有无、期初/期末列的排列方式、千分位逗号。同时通过 80+ 手写科目映射表 + difflib 模糊匹配（阈值 0.75），把五花八门的中文科目名映射到标准字段。这是整个系统的核心壁垒——**没有现成的开源库能处理中国中小企业的非标报表**。

**难点二：中国会计准则的指标体系自建**

中国《企业会计准则》与国际财务报告准则（IFRS）有诸多差异，现有开源财务计算库（如 FinanceToolkit）无法直接适配：

| 指标 | 国际标准公式 | 峤远中国准则版 |
|------|------------|--------------|
| **毛利率** | `(Revenue - COGS) / Revenue` | `(营收 - 营业成本 - 税金及附加) / 营收` |
| **速动比率** | `(流动资产 - 存货 - 预付) / 流动负债` | `(流动资产 - 存货) / 流动负债`（中小企业无常持有价证券） |
| **利息保障倍数** | `EBIT / 利息支出` | `(利润总额 + ∣财务费用∣) / ∣财务费用∣`（不单独列利息支出） |
| **EBITDA** | `EBIT + 折旧 + 摊销` | 三段式替代：`EBIT → +D&A（有则加）→ +减值（有则加）→ 兜底估算` |
| **Altman Z-score** | X5 系数 = 1.0（美股原版） | X5 系数 = 0.999（中国学者修正），判据 1.81 / 2.99 |

全部 7 大类 20+ 指标经财务专业人员审核确认（审核人：峤远，财务+税务+融资视角）。

**难点三：LLM 仅解读不推导的架构设计**

峤远的 LLM（DeepSeek）**不得接触原始报表数据**，只能接收结构化后的指标摘要。所有数值在计算引擎中确定，LLM 唯一职责是用自然语言解读这些数字——"不能编造数据、不能自行推导财务关系、不能假装知道行业基准"。Prompt 设计包含明确的反幻觉指令（temperature=0.3）。

**难点四：数据质量自动门禁**

在上传的报表进入分析管线之前，四道质量校验门禁自动筛查：
- 勾稽关系校验：资产 ≠ 负债 + 权益（偏差超过 5%）→ 阻断，报告中首句提示"数据存在异常"
- 负值检测：货币资金/存货为负 → 阻断（报表本身有问题）
- 逻辑校验：零营收却有净利润 → 阻断
- 极值检测：资产负债率 > 100%、毛利率 > 95% → 警告

确保分析结论建立在数据本身可靠的基础上。

## 四、收费模式：按次付费，可推广分账

### 4.1 为什么是"按次"？

市面上同类产品的收费模式无一例外是**年费/会员制**：

| 产品 | 收费模式 | 参考价格 |
|------|---------|---------|
| 金蝶·金钥财报 | C端免费（上市公司数据）/ 企业版付费 | 企业版价格未公开 |
| FineReport / FineBI | 年费 | 数万-数十万/年 |
| 吉贝克·小吉财金 | 定制报价 | 未公开 |
| 慧算账 / 大账房 | 年费代账 | 2,000-5,000 元/年（仅做账报税） |

**按次收费对中小企业主意味着什么？**

- 🔑 **零门槛**：今天想看报表，今天就付一次钱，不需要全年承诺
- 🔑 **不浪费**：中小企业一年可能只需要做 2-4 次深度财务分析（季末/年末/融资前），没必要全年订阅
- 🔑 **算得清**：每次 20-30 元，老板一眼就懂

| **融资能力专项** | 标准版 + 融资视角深度分析 + 银行关注指标专项解读 | ¥30 / 次 |

## 五、市场时机

### 5.1 为什么是现在？

1. **代账行业升级刚需**：全国几十万家代账公司在低价竞争下利润越来越薄，亟需增值服务提升客单价——财务分析是最自然的升级方向
2. **中小企业融资前置需求**：银行对中小企业贷款的财务审核越来越严，老板在见银行之前需要先"体检"
3. **金税四期推动财务规范化**：税务监管趋严，企业主动或被动的财务合规需求爆发
4. **AI 让"请不起 CFO"成为过去式**：过去只有大企业用得起的财务分析，现在 AI 可以用 ¥20/次提供
5. **赛道尚无同类**：没有第二个产品专为中国中小企业非标报表做 AI 财务分析

### 5.2 竞争壁垒

| 壁垒类型 | 说明 |
|---------|------|
| **科目映射壁垒** | 80+ 手写中国会计准则科目映射 + 模糊匹配，覆盖中小企业常见科目变体 |
| **格式适配壁垒** | 6 种真实企业报表格式自动检测和解析，不做假设，真实测试通过 |
| **公式壁垒** | 20+ 指标全部自研中国准则适配版，非翻译国际公式 |
| **数据壁垒** | 8 份真实企业报表验证，未来持续积累形成行业对标库 |
| **合规壁垒** | 用户授权协议门禁 + 全本地计算 + 仅发结构化摘要给 AI |
