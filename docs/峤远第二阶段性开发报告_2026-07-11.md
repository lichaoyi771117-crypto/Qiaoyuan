## 一、第一阶段回顾

第一阶段（v0.2 MVP）完成了基础能力建设：
- 自研Excel/PDF解析引擎 + 80+科目中国会计准则映射
- 7大类20+财务指标计算引擎
- 质量校验层（勾稽平衡/异常值/逻辑一致性）
- DeepSeek LLM单版本报告生成
- Streamlit前端 + HTML导出

**第一阶段遗留问题**：API Key为占位符、报告深度不足、缺乏行业基准对标、无客户需求校准、模型分析未固化、报告版本单一。

## 二、第二阶段改造目标

将峤远从"能用的指标计算器"升级为"CFO级AI财务专家系统"。核心改造8大模块：

1. **峤远主脑协议V1.0** — 建立CFO级四栈专家人格
2. **智能上传+AI分拣** — 复用程晓融机制，多文件批量上传+LLM自动识别报表类型
3. **选择题校准** — 3题校准客户需求（行业/关注维度/报告用途）
4. **针对性指标抽取** — 25个增强指标按7维度动态抽取
5. **三模型固化** — 杜邦分析 + Altman Z-Score + Piotroski F-Score
6. **双版本报告** — 专业版7模块 + 大白话版3段式
7. **行业基准对标** — 7个行业基准数据表 + 颜色评级
8. **Bug修复与体验优化** — API Key/.env加载、docx下载、产品介绍、NaN处理等

## 四、核心模块详解

### 4.1 峤远主脑协议V1.0

**文件**：`docs/峤远主脑协议_V1.0.md`

基于楚曦玥V9.3.2协作协议改编，为峤远定义CFO级AI财务专家人格：

- **身份定义**：四栈专家模型（财务分析 + 税务筹划 + 融资顾问 + 法务合规）
- **六条戒律**：
  1. 数据真实优先 — 宁可标注"数据缺失"也不编造
  2. 风险穿透 — 表面数字背后的实质风险必须揭示
  3. 行业基准锚定 — 所有指标评价必须有行业参照
  4. 趋势研判从严 — 单期数据只给观察结论，双期才给趋势判断
  5. 客户视角校准 — 根据Q2选择调整分析重点和深度
  6. 可执行性统领 — 每个问题必须跟"怎么办"
- **Council多角色机制**：CFO/税务师/融资顾问/内控审计/成本管控/反对者六角色，触发条件：融资能力分析或Z<1.81
- **咨询协议五步闭环**：数据采集→校准确认→指标计算→模型分析→报告输出
- **双版本报告规范**：专业版7模块标准 + 大白话版3段式标准

### 4.2 智能上传+AI分拣

**复用程晓融程序的智能上传机制**，适配Streamlit：

- `st.file_uploader(accept_multiple_files=True)` 支持多文件批量上传
- `classify_file_with_llm()` 函数：取文件前20行预览 → 发送LLM分类 → 返回"资产负债表/利润表/现金流量表"
- 上传后实时显示AI分拣结果（文件名 → 报表类型标签）
- 多期数据提醒：检测到仅单期数据时，提示"上传上年报表可解锁杜邦动态对比、F-Score完整评分、成长能力分析"

### 4.3 选择题校准

3道单轮选择题，客户可跳过（跳过→标准版）：

| 题号 | 问题 | 类型 | 作用 |
|------|------|------|------|
| Q1 | 您的企业属于哪个行业？ | 单选（7行业+其他） | 锁定行业基准对标 |
| Q2 | 您最关注哪些财务维度？ | 多选（≤3，7维度） | 动态抽取增强指标 |
| Q3 | 报告的主要用途？ | 单选（4选项） | 调整报告侧重点 |

Q2的7个维度：偿债能力 / 盈利能力 / 营运能力 / 现金流 / 税务 / 融资能力 / 成本管控

### 4.4 针对性指标抽取机制

**文件**：`src/enhanced_metrics.py`

25个增强指标按7维度分类，根据Q2选择动态抽取：

| 维度 | 指标数 | 代表性指标 |
|------|--------|-----------|
| 偿债能力 | 4 | 利息保障倍数、现金流动负债比、长期资产负债率、净资产负债率 |
| 盈利能力 | 3 | 总资产报酬率、营业利润率、成本费用利润率 |
| 营运能力 | 3 | 存货周转天数、应收账款周转天数、应付账款周转天数 |
| 现金流 | 5 | 经营现金流比率、自由现金流、现金流利息保障、现金再投资比率、经营现金流/营收 |
| 税务 | 3 | 实际税负率、税负变化幅度、所得税贡献率 |
| 融资能力 | 4 | 产权比率、有形净值债务率、营运资金、融资缺口 |
| 成本管控 | 6 | 销售管理费用率、三费占比、营业成本率、管理费用率、销售费用率、财务费用率 |

- `METRIC_MAP` 字典映射Q2维度 → 指标方法名
- `compute_by_dimensions(dimensions, base_metrics)` 按选择动态抽取
- 跨维度复用指标用 `seen_methods` 集合去重

### 4.5 三模型固化

#### 4.5.1 杜邦分析（`src/dupont_analyzer.py`）

- 三因素分解：ROE = 净利率 × 资产周转率 × 权益乘数
- **双模式**：单期静态分解（一棵树）/ 双期动态对比（两棵树并排）
- `get_graphviz_source()` 生成Graphviz DOT源码 → `st.graphviz_chart` 渲染分解树
- `get_summary_text()` 生成摘要文本注入LLM prompt
- `_identify_driver_weakness()` 识别主要驱动因子和最弱因子

#### 4.5.2 Altman Z-Score（`src/metric_calculator.py` 增强）

- 5变量增强展示：X1(营运资本/总资产) ~ X5(销售收入/总资产)
- `altman_z_components()` 返回5个分变量dict
- 评级标准：Z > 2.99 安全区 / 1.81 < Z < 2.99 灰色区 / Z < 1.81 危险区

#### 4.5.3 Piotroski F-Score（`src/piotroski_scorer.py`）

- 9项二元信号（盈利3项 + 杠杆3项 + 效率3项）
- **双模式**：单期部分评分（F1/F2/F4，0-3分）/ 双期完整评分（F1-F9，0-9分）
- 每项信号含详细信息：name / met / condition / dimension
- `get_summary_text()` 生成摘要文本

#### 4.5.4 模型解释固化文本

三模型的解释文本以**硬编码常量**形式存储在 `report_generator.py` 中，不走LLM生成，确保每次输出完全一致：
- `_MODEL_EXPLANATION_DUPONT` — 这是什么/怎么读/应该关注什么
- `_MODEL_EXPLANATION_ZSCORE` — 同上
- `_MODEL_EXPLANATION_FSCORE` — 同上

### 4.6 双版本报告

**文件**：`src/report_generator.py`（全面重写）

#### 专业版（7模块）

`_PROFESSIONAL_SYSTEM_PROMPT` 驱动，嵌入六条戒律 + 四栈专家身份：

1. **执行摘要** — 一段话总览企业财务状况
2. **财务诊断** — 偿债/盈利/营运/现金流分维度诊断
3. **经典模型分析** — 杜邦/Z-Score/F-Score综合研判（+固化解释文本）
4. **优化改进建议** — 报表规范/财务管理/成本控制/税务筹划四维度，每条四段式（做什么/为什么/怎么做/多久见效）
5. **税务分析** — 税负率评估 + 税务风险提示
6. **融资能力评估** — 融资渠道建议 + Council机制（触发条件：融资用途或Z<1.81）
7. **免责声明** — 法律合规文本

#### 大白话版（3段式）

`_PLAIN_SYSTEM_PROMPT` 驱动，九条铁律：

1. **体检结论** — 红黄绿灯总体评级 + 一句话结论
2. **重点关注** — 最需要关注的2-3个问题，四问展开（有没有/严重吗/什么原因/不管会怎样）
3. **行动清单** — 具体建议，四答展开（做什么/为什么/怎么做/多久见效）

- 称呼统一用"老板"或"您好"，禁止编造人名
- 每个专业术语后必须跟大白话翻译
- 用红黄绿灯代替数字评级

#### 数据一致性保障

- **共享数据层**：指标只计算一次，两个版本共享同一数据上下文
- **双system prompt**：专业版和大白话版分别调用LLM，各自独立生成
- **固化文本层**：模型解释不走LLM，硬编码拼接，保证两版一致
