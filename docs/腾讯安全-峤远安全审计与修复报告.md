## 三、详细修复说明

### H-1 Git remote URL 嵌入明文 GitHub token

**问题**：`.git/config` 中 origin URL 包含 `ghp_` 开头的 classic token，每次 push 都会以 URL 形式发送 token。

**修复**：将 remote URL 重置为无 token 的 HTTPS 地址。

```bash
git remote set-url origin https://github.com/lichaoyi771117-crypto/Qiaoyuan.git
```

---

### H-2 真实企业财务报表被 git 跟踪

**问题**：`test documents/` 下包含多家真实企业的财务报表（`.xls/.xlsx`）及生成的 HTML 分析报告，已随 git 历史推送到 GitHub。

**影响**：企业财务敏感数据泄露风险。

**修复**：

```bash
git rm -r --cached "test documents"
```

并在 `.gitignore` 中增加：

```gitignore
test documents/
Key.txt
*token*.txt
```

本地文件仍保留，但不再进入版本控制。

---

### H-3 源代码硬编码真实 PII

**问题**：

- `app.py:135-137` 硬编码联系人余磊、电话 `13987671259`、邮箱 `425448719@qq.com`
- `app.py:650` 和 `src/report_generator.py:175` 硬编码开发人员姓名

**修复**：将真实 PII 替换为"[请联系客服获取]"，将开发人员信息替换为免责声明。

---

### M-1 / M-2 文件上传大小限制与类型校验

**问题**：`st.file_uploader` 仅通过扩展名过滤，无单文件大小、总大小、Magic Bytes 校验。

**修复**：新增 `src/app_security.py` 提供：

- `MAX_SINGLE_FILE_SIZE = 20 MB`
- `MAX_TOTAL_UPLOAD_SIZE = 50 MB`
- `MAX_UPLOAD_FILES = 10`
- Excel Magic Bytes 校验：`.xlsx` 必须以 `PK\x03\x04` 开头；`.xls` 必须以 `\xd0\xcf\x11\xe0` 开头

代码位置：

```python
from src.app_security import validate_uploaded_files

ok, err, valid_files = validate_uploaded_files(uploaded_files)
if not ok:
    st.error(err)
    st.stop()
```

---

### M-3 LLM 调用未设置 timeout

**问题**：`app.py:classify_file_with_llm` 和 `src/report_generator.py:_call_llm` 均无 timeout，网络异常时可能长时间挂死。

**修复**：

```python
# app.py
response = client.chat.completions.create(..., timeout=LLM_TIMEOUT_SECONDS)

# src/report_generator.py
response = self._client.chat.completions.create(..., timeout=120)
```

---

### M-4 错误信息泄露

**问题**：`src/report_generator.py:_call_llm` 原代码将 `str(e)` 直接返回给前端。

**修复**：异常统一返回"[模型调用失败，请稍后重试]"，内部异常仅写入日志。

---

### M-5 缺少请求限流

**问题**：用户可重复点击"一键生成"，导致 DeepSeek API 额度被快速消耗。

**修复**：基于 `st.session_state` 实现 session 级限流：

- 文件分类：`10 次 / 60 秒`
- 生成报告：`5 次 / 60 秒`

---

### M-6 API Key 启动未强制校验

**问题**：`DEEPSEEK_API_KEY` 为空字符串时程序仍可运行，只在调用 LLM 时才失败。

**修复**：在 `st.set_page_config` 之后立即校验：

```python
from src.app_security import validate_api_key
validate_api_key(DEEPSEEK_API_KEY)
```

未配置时给出明确提示并停止运行。
