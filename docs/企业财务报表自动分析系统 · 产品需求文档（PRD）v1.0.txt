- **版本**：v1.0

## 2. 项目背景与目标
霖信莯咨询是一家以助贷为主业、正向企业财务税务咨询和融资顾问转型的小型公司。客户多为中小企业，普遍存在财务报表不规范、财务结构不合理、融资困难等问题。公司计划开发一款在网页端运行的企业财务报表自动分析程序，实现以下目标：
- 客户自助上传财务报表（年报/半年报为主，兼容季报、月报）
- 系统自动计算核心财务指标，生成结构化解读
- 发现财务体系的异常与优化空间
- 围绕“财务优化、融资增信、降本增效”等客户目标，输出可落地的建议报告
- 最终交付物为一份专业分析报告，作为高附加值咨询服务的引流产品或独立出售

## 3. 用户与场景
- **主要用户**：霖信莯咨询的客户（中小企业主、财务人员）
- **使用场景**：通过网页上传 PDF/Excel/图片格式的财务报表，选择分析目标（财务健康检查/融资能力评估/降本增效），数分钟内获得分析报告
- **辅助用户**：霖信莯咨询内部顾问，可基于系统生成的报告进行人工复核与深化服务

## 4. 核心功能需求

### 4.1 报表输入与识别
- 支持上传格式：PDF、Excel（.xls/.xlsx）、图片（PNG/JPG）
- 自动识别报表类型：资产负债表、利润表、现金流量表
- 科目提取与标准化映射（关键：兼容中国会计准则下的常见科目名称）
- 报表勾稽关系校验（如资产=负债+权益，不平需标红警告）

### 4.2 财务指标计算
系统需自动计算以下**7大类、20+核心指标**，并支持按行业细分对比（初期可配置静态基准值）：

| 大类 | 指标名称 | 计算公式 | 融资/尽调关注点 |
|------|----------|----------|------------------|
| **偿债能力** | 流动比率 | 流动资产 / 流动负债 | 银行短期偿债能力红线（通常要求>1） |
| | 速动比率 | (流动资产-存货) / 流动负债 | 剔除存货后的即时偿债能力 |
| | 资产负债率 | 总负债 / 总资产 | 杠杆水平，过高则融资困难 |
| | 利息保障倍数 | EBIT / 利息费用 | 利润覆盖利息的能力，银行重点关注 |
| **盈利能力** | 毛利率 | (营收-营业成本) / 营收 | 产品/服务基本盈利空间 |
| | 净利率 | 净利润 / 营收 | 整体盈利质量 |
| | ROE（净资产收益率） | 净利润 / 平均净资产 | 股东回报，股权融资核心指标 |
| | ROA（总资产收益率） | 净利润 / 平均总资产 | 资产运用效率 |
| | EBITDA 率 | EBITDA / 营收 | 适合重资产企业估值与授信参考 |
| **营运效率** | 存货周转天数 | 365 / (营业成本/平均存货) | 资金压在存货上的时间 |
| | 应收账款周转天数 | 365 / (营收/平均应收账款) | 回款速度，助贷客户最常见痛点 |
| | 总资产周转率 | 营收 / 平均总资产 | 每一元资产创造的收入 |
| **成长能力** | 营收增长率 | (本期营收-上期营收)/上期营收 | 业绩扩张速度 |
| | 净利润增长率 | (本期净利润-上期净利润)/上期净利润 | 盈利成长性 |
| | 资本积累率 | (期末权益-期初权益)/期初权益 | 资本保全与增长 |
| **现金流** | 经营现金流/净利润 | 经营活动现金流净额 / 净利润 | 利润含金量，税务与银行的核心指标 |
| | 自由现金流 | 经营活动现金流净额 - 资本支出 | 企业可自由支配的现金 |
| **税务指标** | 实际税率 | 所得税费用 / 利润总额 | 税负合理性，税务筹划依据 |
| | 综合税负率 | 各项税费支出 / 营收 | 整体税负水平 |
| **财务预警** | Altman Z-score | 适用于非上市制造业/服务业公式 | 破产风险预测，技术性破产预警 |

**说明**：若财报数据不全（如未披露EBITDA），系统应使用替代公式估算或标记为不可计算。

### 4.3 分析报告生成
报告需包含以下模块，文字内容需结合大语言模型（LLM）生成自然语言解读：
1. **财务数据摘要**：关键科目一览，自动校验勾稽关系，异常项标红
2. **指标雷达图与行业对标**：将计算结果与行业基准值对比（可先使用中国统计局公开行业均值，或允许自定义基准）
3. **异常诊断**：指标显著偏离合理区间的可能原因（由LLM结合财务逻辑解释）
4. **优化建议**：按客户选择的目标（财务优化/融资/降本增效）生成具体可操作建议，例如：
   - *“应收账款周转天数为187天，远高于制造业中位数62天，建议收紧信用政策或考虑应收账款保理以快速回笼资金。”*
5. **税务风险提示**：费用结构异常、税负率突变、关联交易迹象等
6. **免责与合规声明**：提示本报告为系统自动生成，不构成专业意见，重大决策需咨询专业顾问

### 4.4 网页端交付
- 简洁的 Web 界面：上传报表 → 选择分析目标 → 等待处理 → 在线阅读/下载 PDF 报告
- 响应式设计，支持 PC 和移动端

## 5. 技术要求
- **技术栈倾向**：Python 后端 + 轻量 Web 框架（Streamlit / Gradio / Flask）
- **数据处理**：Pandas, Tabula (PDF解析), Camelot, OpenPyXL
- **指标计算**：模块化的财务计算引擎，公式可配置
- **LLM 集成**：OpenAI API 或本地部署模型，用于生成分析文本
- **数据库**：SQLite 或 PostgreSQL，存储用户记录与基准数据
- **部署**：支持 Docker，可部署至云服务器或本地

## 6. 关于国内外财务指标差异的说明
多数财务指标（如流动比率、ROE）在全球范围内通用，**但在具体应用中存在以下差异**：
- **科目名称与披露格式**：中国采用《企业会计准则》，科目名称、报表格式与 IFRS/US GAAP 有区别。例如“税金及附加”、“营业外收支”的披露位置不同。
- **行业基准值**：国内行业均值与欧美企业差异较大，参考标准不能直接套用国外公开数据库。
- **税务指标**：中国的税种（增值税、企业所得税等）与税率结构独具特色，部分指标（如综合税负率）需根据国内税法设计。
- **财务预警模型**：通用 Z-score 直接应用于国内企业需要修正参数。

**因此，在选择开源项目时，需重点关注是否支持中国准则下的科目映射，或具备灵活的科目配置能力。**
